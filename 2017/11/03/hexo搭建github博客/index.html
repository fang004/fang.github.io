hexo搭建github博客

导读

用过很多记录工具，印象笔记、oneNote、有道云笔记等等，各坚持一段时间，换了电脑后文章都不知道丢到哪里去了。搭建个人网站，从WordPress到hexo。
以下是我的使用感受，带有强烈的个人主观偏见。

  • 印象笔记
    • 我使用这个比较早，0几年的时候就用了，用过一段时间后转成了oneNote
    • 优点
      • 使用时间较短，过早放弃，总结不出来了。
    • 缺点
      • 长的太丑（这是最主要的）
      • 编辑器不满足
      • 免费只支持2个设备同步
  • oneNote
    • 用过几年，因为从windows换成了mac，而放弃了它，转入有道云笔记怀抱
    • 优点
      • 与word``excel是一个爸爸，界面熟悉，上手快
      • 一个爸爸的软件兼容很好
      • 编辑器强大，适合平时记录
      • 搜索方便
    • 缺点
      • 同步太痛苦
      • 太慢
      • 不支持markdown，无法归档成html
      • mac上臃肿
  • 有道云笔记（我放弃的原因是公司禁了163，然后就无法同步了……）
    • 虽然缺点有许多，编辑工具抵不上oneNote，但同步方便啊，一直在使用，直到公司禁了163帐号，转入了hexo
    • 优点
      • 同步快
      • 分享快，直接生成网址，只要有网都可以访问。
    • 缺点
      • mac上使用有时会崩溃
      • 编辑时时光标定位不准
      • 图片与文字之间的排版很不爽，没有oneNote强大
  • WordPress
    • 优点
      • 插件多，功能强大
    • 缺点
      • 太重了，对于个人归档来说，不太适合
      • 服务器、数据库依赖，需要服务器支持
  • hexo
    • 刚入坑，正处于情人眼中出西施的阶段
    • 优点
      • github不倒，你的资料永远在
      • 对于前端的我来说，不需要和服务器做交互，操作简单
      • 主题diy简单，颜控的我来说很喜欢
      • 有中文api文档
      • 支持 Markdown解析文章，在几秒内，即可利用靓丽的主题生成静态网页。
      • 快速与github主页绑定
    • 缺点
      • 图片资源需要自己处理，目前我是把图片推到github上面使用，正好我的hexo也是在github上面，也可使用七牛托管
      • 需要一些前端基础

安装

前提

  • Node.js
  • Git

安装hexo库

npm install -g hexo-cli

建站

步骤

  • 逐条运行以下命令，其中<folder>是你库所在的文件夹名称，这只是一个标识，自己认识就行了，没特殊要求（我的库文件夹名称为fang.github.io）
  • 每条命令含义
    • 初始hexo
    • 进入到库所在的文件夹
    • 安装package.json所需要的包
hexo init <folder>
cd <folder>
npm install

目录下各文件作用

  • 完成后的文件夹目录如下
  • 各文件作用
    | 文件名 | 作用 |
    | —- | —- |
    | _cofnig.yml | 网站的配置信息，这里配置hexo网站的大部分信息 |
    | package.json | 项目所需要的插件目录，通过npm install安装，如果你是前端，这个应该很熟悉了|
    | scaffolds | 模板文件夹。新建文件的时候，hexo会根据这里面的模板来新建文件夹 |
    | source | 新增编辑的md文件都存在这里，除了_posts文件夹，其他_开头的隐藏文件或文件均，在hexo generate时都会被忽略。Markdown 和 HTML 文件会被解析并放到 public 文件夹，而其他文件会被拷贝过去。 |
    | source/_drafts | 草稿所在的目录 |
    | source/_posts | 文章所在的目录 |
    | themes | 主题文件夹（各个子文件夹为主题名称，子文件夹包括样式、脚本、模板，hexo会根据主题来生成静态文件|

浏览你的页面

  • 生成静态文件
hexo generate
  • 启动服务器
hexo server
  • 在浏览器中通过http://localhost:4000/来访问（4000端口号是默认的，在启动服务器的时候，可以带参数修改）
    • 通过通过服务器浏览到页面，代表建站完成。
    • 如果页面和下图不一样不要紧，只是选择的主题不一样，下的的主题是我自己写的，后面会讲到如何开发主题。

关联github

注册github帐号（已有跳过）

下面简单的说下注册步骤，想详情的可以百度搜索：如何注册github

输入账号、邮箱、密码,然后点击注册按钮.

初始设置：注册完成后,选择免费账号完成设置。

  • 验证邮箱：打开你注册时填的邮箱，确认发送给确认邮件。（确认验证邮箱后，生成的个人主页才会被接收和发布）

创建页面仓库

  • 注意，这个仓库的名字必须与帐号一致，.github.io后缀不可修改，必须。

设置你的项目

  • 因为我已经有了github帐号，前几步注册的github帐号激活不成功，无法发布，换成了现在用的帐号。
  • 项目设置在顶部右侧，如下图
  • 主页的项目名称是固定的，前缀为用户名，后面为github.io
  • 首页的分支为master
  • 其他的设置使用默认值即可，主题等等我们可以不用care，首页的项目以hexo搭建，文件会被替换。

修改配置文件config.yml

  • repo地址可以从github项目里面直接复制，见下图
deploy:
  type: git
  repo: https://github.com/fang004/fang004.github.io.git
  branch: master

  • 安装插件

    npm install hexo-deployer-git –save

  • 同步文件至github

    hexo deploy

在浏览器中输入https://fang004.github.io即可浏览你布置的hexo项目

  • github首页看到的页面，除了网址不一样，其他的和你本地看的到是一致的。

github相关设置说明

license设置

  • 阮一峰博文如何选择开源许可证？可作为简单参考

仓库分支介绍

  • 仓库的分支，个人主页是master分支，其他的项目是gh-pages分支，gh是github的简称
    • 比如你另外新建了一个hello的分支，相应的访问路径为https://fang004.github.io/hello
    • 其中fang004和你的用户名一致，hello是你的项目名

常用命令

hexo clean #清除缓存和已生成的静态文件
hexo list #列出风锌资料

hexo --draft #显示草稿

hexo new "postName" #新建文章
hexo new page "pageName" #新建页面
hexo new draft "new draft" #新建草稿
hexo publish [layout] <filename> #草稿变成文章
hexo generate #生成静态页面至public目录
hexo server #开启预览访问端口（默认端口4000，'ctrl + c'关闭server）
hexo deploy #将.deploy目录部署到GitHub
hexo help # 查看帮助
hexo version #查看Hexo的版本

    # 然后依次执行
    hexo g
    hexo d
